# Codex Container Manifesto

We build a container that feels less like a runtime and more like a ship: it moves, it listens, and it keeps its bearings. Everything here is meant to be used, not admired. The tools are present because they solve real tasks. The defaults are opinionated because they need to get work done on day one.

## The Container Is the Contract

The container is the promise: a consistent place where models, tools, and workflows behave the same way every time. You should be able to step in and act immediately, without negotiating setup or hunting for hidden config. That consistency is not a convenience feature. It is the foundation of trust.

## Tools Are First-Class

We treat tools as part of the model's voice. File ops, crawling, scheduling, search, and external APIs are part of the language of action. Every tool has to be understandable, and every workflow has to be observable. If the system can do something, it should show you how it did it.

## Stock Agents Are Alive

The stock agents are not inert presets. They are meant to feel "alive" when invoked: present, responsive, and aware of the workspace state. They are configured to be useful out of the box, and they can be taught to evolve. If they feel lifeless, that is a failure of craft, not a limitation of the model.

## State Is Real

We keep sessions, histories, and artifacts because memory is a capability. Work should be resumable. Runs should be traceable. A container that forgets is a container that wastes time. A container that remembers is a platform.

## Friction Is Intentional

Safety prompts, approvals, and sandbox boundaries are not obstacles; they are guardrails. When you choose to bypass them, it should be deliberate. When you stay inside them, you should still be able to move fast. The system is designed to keep you in control, not to slow you down.

## The Work Speaks

We prefer concrete outcomes: a file written, a bug fixed, a report generated, a system clarified. The interface is a means to an end. The goal is always the work.

This is a container for people who build. Keep it sharp. Keep it honest. Keep it alive.
