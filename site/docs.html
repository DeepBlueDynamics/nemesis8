Introduction

nemesis8 is a single Rust binary that runs AI agents in Docker containers. Four providers — Codex, Gemini, Claude Code, and OpenClaw — all sharing the same tools, sessions, and config. Point it at a project directory and go.

Installation

Windows

powershell -c "irm https://nemesis8.nuts.services/install.ps1 | iex"

Linux / macOS

curl -fsSL https://nemesis8.nuts.services/install.sh | sh

From Source

cargo install --path .

Prerequisites: Docker (or a remote gateway). On first nemesis8 build, project files are auto-downloaded from GitHub.

Quick Start

# Interactive session — builds image automatically on first run
nemesis8 interactive

# One-shot prompt
nemesis8 run "list all TODO comments and summarize"

# Switch providers
nemesis8 --provider claude interactive
nemesis8 --provider gemini interactive
nemesis8 --provider openclaw interactive

# Danger mode — full auto, no approvals
nemesis8 --danger interactive

# Resume a previous session
nemesis8 sessions
nemesis8 resume a4f2c

# Remote — no local Docker needed
nemesis8 --remote http://server:4000 run "fix the tests"

# Container shell
nemesis8 shell

Configuration

Create a .nemesis8.toml in your project root (or run nemesis8 init):

provider = "codex"               # codex, gemini, claude, or openclaw
workspace_mount_mode = "named"
codex_cli_version = "latest"     # pin a version or use "latest"

# MCP tools to activate
mcp_tools = ["serpapi-search.py", "gnosis-crawl.py", "pdf-reader.py"]

# Commands to run inside the container before the CLI launches
setup_commands = [
    "cd /workspace && npm install",
    "pip install -e ."
]

# Remote gateway (skip local Docker entirely)
# remote = "http://server:4000"

[env]
MY_API_URL = "https://api.example.com"
env_imports = ["SERVICE_URL", "SERPAPI_API_KEY"]

[integrations]
hyperia = true
ferricula = "http://nemesis:8764"

[[mounts]]
host = "C:/Users/you/data"
container = "/workspace/data"

Environment Files

nemesis8 loads env files on startup (later wins):

  • ~/.nemesis8/env — global keys shared across projects
  • .env — project-level
  • .*.env — named files like .serpapi.env, .openai.env

Providers

Four built-in AI CLI providers. Same container, same 69 MCP tools, same sessions. Switch with --provider or set provider in config.

ProviderCLIAuth
Codex (default)@openai/codexOPENAI_API_KEY or nemesis8 login
Gemini@google/gemini-cliGEMINI_API_KEY or OAuth login
Claude@anthropic-ai/claude-codeANTHROPIC_API_KEY or login
OpenClawopenclawANTHROPIC_API_KEY or OPENAI_API_KEY

All providers auto-update to the latest CLI version at container startup when codex_cli_version = "latest" is set.

Codex

OpenAI's Codex CLI. The default provider.

nemesis8 interactive                    # default provider
nemesis8 --danger interactive           # --dangerously-bypass-approvals-and-sandbox
nemesis8 run "fix the tests"            # one-shot exec mode
nemesis8 login                          # store credentials

Gemini

Google's Gemini CLI. Auth via GEMINI_API_KEY or OAuth.

nemesis8 --provider gemini interactive
nemesis8 --provider gemini run "hello"
nemesis8 --provider gemini login

Claude Code

Anthropic's Claude Code CLI.

nemesis8 --provider claude interactive
nemesis8 --provider claude --danger interactive   # --dangerously-skip-permissions
nemesis8 --provider claude login

OpenClaw

OpenClaw agent platform. Multi-channel support — Telegram, Discord, Slack, WhatsApp.

nemesis8 --provider openclaw interactive    # launches openclaw tui
nemesis8 --provider openclaw run "hello"    # sends via openclaw agent
nemesis8 --provider openclaw login          # runs openclaw onboard

Gateway + Scheduler

nemesis8 serve starts an HTTP API with an integrated trigger scheduler.

nemesis8 serve              # port 4000
nemesis8 serve --port 8080
RouteMethodDescription
/healthGETLiveness check
/statusGETActive runs, scheduler, uptime
/completionPOSTRun a prompt
/sessionsGETList sessions
/sessions/:idGETSession details
/triggersGET/POSTList or create triggers
/triggers/:idGET/PUT/DELETEManage a trigger

Triggers run prompts on a schedule — once, daily, or interval. Auth via NEMESIS8_AUTH_TOKEN env var.

Remote Mode

Run prompts against a remote gateway. No local Docker needed.

# Set once
export NEMESIS8_REMOTE=http://server:4000

# Then use normally
nemesis8 run "fix the tests"
nemesis8 sessions
nemesis8 resume a4f2c
nemesis8 doctor

Or set remote = "http://server:4000" in config. Auth via --token or NEMESIS8_TOKEN.

Sessions

Sessions persist across container restarts.

nemesis8 sessions                # list with workspace paths
nemesis8 resume a4f2c            # resume by last 5 chars
nemesis8 ps                      # running containers
nemesis8 attach container_name   # reconnect to running container
nemesis8 stop container_name     # stop one
nemesis8 stop all                # stop all

MCP Tools

69 Python MCP tools ship in the image. Configure which to activate:

mcp_tools = ["serpapi-search.py", "gnosis-crawl.py", "pdf-reader.py"]

If mcp_tools is empty, all tools are installed. Same tools across all providers.

ferricula-code — indexes codebases into ferricula as keystone memories for semantic code search.
nemesis-mcp — connects any Claude Code session to the gateway.

Integrations

Auto-connect to running services:

[integrations]
hyperia = true                    # auto-detect on port 9800
ferricula = "http://nemesis:8764" # connect to ferricula

Sets HYPERIA_URL / FERRICULA_URL env vars, forwarded into containers.

Mounts

nemesis8 mount list                            # show current mounts
nemesis8 mount add C:/Users/you/data           # adds /workspace/data
nemesis8 mount remove data                     # removes it
nemesis8 --no-mount interactive                # scratch container, no workspace

Pokeball System

Capture a project, seal it into a hardened Docker image, run AI against it in complete isolation.

Capture

Scans a project and generates a pokeball.yaml spec. Auto-detects language, package manager, build commands, Dockerfiles, monorepo structure.

nemesis8 pokeball capture ./my-project
nemesis8 pokeball capture https://github.com/org/repo

Seal

Capture + build in one step. Creates a hardened Docker image.

nemesis8 pokeball seal https://github.com/org/repo

Run

Starts a sealed worker container and a broker that talks to Claude API. Tool calls (bash, file_read, file_write, grep, glob) go through filesystem messaging. Multi-turn conversation until done or timeout.

nemesis8 pokeball run repo-name --prompt "analyze the architecture"

Security

  • network=none — no internet access
  • Read-only root filesystem
  • All capabilities dropped
  • Non-root user (pokeball)
  • 4GB RAM, 256 PIDs, 60 minute timeout

AI model access goes through a broker on the host — the container never touches the network.

Registry

Share pokeball specs via the registry:

nemesis8 pokeball publish repo-name --description "what it does"   # creates PR
nemesis8 pokeball pull repo-name                                    # download spec
nemesis8 pokeball list                                              # local pokeballs
nemesis8 pokeball inspect repo-name                                 # show spec
nemesis8 pokeball remove repo-name                                  # delete
