Introduction

nemesis8 is a single Rust binary that runs AI agents in Docker containers. Four providers — Codex, Gemini, Claude Code, and OpenClaw — all sharing the same tools, sessions, and config.

Point it at a project directory. It builds the image, installs MCP tools, forwards credentials, and launches the CLI. Sessions persist. Containers are isolated. One binary, total control.

nemesis8 interactive                           # start a session
nemesis8 --provider claude interactive         # switch brains
nemesis8 --danger interactive                  # full auto mode
nemesis8 pokeball run hyperagents --prompt "analyze the meta-agent loop"

Installation

Windows

powershell -c "irm https://nemesis8.nuts.services/install.ps1 | iex"

Linux / macOS

curl -fsSL https://nemesis8.nuts.services/install.sh | sh

From Source

cargo install --path .

Prerequisites: Docker (or a remote gateway). API keys optional — set them if your provider needs them.

On first nemesis8 build, the project files are automatically downloaded from GitHub to ~/.nemesis8/project.

Quick Start

# Run a prompt — builds the image automatically on first run
nemesis8 run "list all TODO comments and summarize"

# Interactive session (full TUI)
nemesis8 interactive

# Switch providers
nemesis8 --provider gemini interactive
nemesis8 --provider claude interactive
nemesis8 --provider openclaw interactive

# Danger mode — skip all approvals and sandboxing
nemesis8 --danger interactive

# Resume a previous session
nemesis8 sessions
nemesis8 resume a4f2c

# Remote mode — no local Docker needed
nemesis8 --remote http://server:4000 run "analyze this codebase"

# Drop into the container shell
nemesis8 shell

Configuration

Create a .nemesis8.toml in your project root (or run nemesis8 init):

provider = "codex"               # codex, gemini, claude, or openclaw
workspace_mount_mode = "named"
codex_cli_version = "latest"     # pin a version or use "latest"

# MCP tools to activate
mcp_tools = ["serpapi-search.py", "gnosis-crawl.py", "pdf-reader.py"]

# Commands to run inside the container before the CLI launches
setup_commands = [
    "cd /workspace && npm install",
    "pip install -e ."
]

# Remote gateway (skip local Docker entirely)
# remote = "http://server:4000"

[env]
MY_API_URL = "https://api.example.com"
env_imports = ["SERVICE_URL", "SERPAPI_API_KEY"]

[integrations]
hyperia = true
ferricula = "http://nemesis:8764"

[[mounts]]
host = "C:/Users/you/data"
container = "/workspace/data"

Providers

nemesis8 ships with four built-in AI CLI providers. Same container, same 69 MCP tools, same session persistence. Just different brains.

ProviderCLIAuth
Codex (default)@openai/codexOPENAI_API_KEY or nemesis8 login
Gemini@google/gemini-cliGEMINI_API_KEY or login
Claude@anthropic-ai/claude-codeANTHROPIC_API_KEY or login
OpenClawopenclawANTHROPIC_API_KEY or OPENAI_API_KEY

Switch at any time with --provider or set provider = "claude" in your config. All providers auto-update to the latest CLI version at container startup.

Codex

OpenAI's Codex CLI. The default provider.

nemesis8 interactive                    # default
nemesis8 --danger interactive           # full auto (--dangerously-bypass-approvals-and-sandbox)
nemesis8 run "fix the tests"            # one-shot exec mode
nemesis8 login                          # store credentials

Set codex_cli_version = "latest" in config to auto-update on every container startup.

Gemini

Google's Gemini CLI.

nemesis8 --provider gemini interactive
nemesis8 --provider gemini run "hello"
nemesis8 --provider gemini login

Auth via GEMINI_API_KEY or OAuth login.

Claude Code

Anthropic's Claude Code CLI.

nemesis8 --provider claude interactive
nemesis8 --provider claude --danger interactive   # --dangerously-skip-permissions
nemesis8 --provider claude login

Auth via ANTHROPIC_API_KEY.

OpenClaw

OpenClaw agent platform with multi-channel support (Telegram, Discord, Slack, WhatsApp).

nemesis8 --provider openclaw interactive    # launches openclaw tui
nemesis8 --provider openclaw run "hello"    # sends via openclaw agent
nemesis8 --provider openclaw login          # runs openclaw onboard

Gateway + Scheduler

nemesis8 serve starts an HTTP API with an integrated trigger scheduler:

nemesis8 serve              # port 4000
nemesis8 serve --port 8080
RouteMethodDescription
/healthGETLiveness check
/statusGETActive runs, scheduler, uptime
/completionPOSTRun a prompt
/sessionsGETList sessions
/sessions/:idGETSession details
/triggersGET/POSTList or create triggers
/triggers/:idGET/PUT/DELETEManage a trigger

Triggers run prompts on a schedule — once at a timestamp, daily at a time, or on an interval. Auth via NEMESIS8_AUTH_TOKEN.

Remote Mode

Run prompts against a remote gateway without Docker installed locally:

# Set once
export NEMESIS8_REMOTE=http://server:4000

# Then use normally
nemesis8 run "fix the tests"
nemesis8 sessions
nemesis8 resume a4f2c
nemesis8 doctor    # shows remote health

Or set remote = "http://server:4000" in your config. Auth token supported via --token or NEMESIS8_TOKEN.

Sessions

Sessions persist across container restarts in ~/.codex-service/.codex/sessions/.

nemesis8 sessions                # list all sessions with workspace paths
nemesis8 resume a4f2c            # resume by last 5 chars of UUID
nemesis8 ps                      # list running containers
nemesis8 attach container_name   # reconnect to a running container
nemesis8 stop container_name     # stop a container
nemesis8 stop all                # stop all nemesis8 containers

MCP Tools

69 Python MCP tools ship in the Docker image, installed at container startup. Configure which ones to activate in .nemesis8.toml:

mcp_tools = ["serpapi-search.py", "gnosis-crawl.py", "pdf-reader.py"]

If mcp_tools is empty or missing, all tools are installed. The same tools work across all four providers.

ferricula-code indexes codebases into ferricula as keystone memories for semantic code search. nemesis-mcp connects any Claude Code session to the gateway.

Environment Files

nemesis8 loads env files on startup (later wins):

  • ~/.nemesis8/env — global keys shared across projects
  • .env — project-level
  • .*.env — named files like .serpapi.env, .openai.env
# .serpapi.env
SERPAPI_API_KEY=abc123

Variables are forwarded to containers automatically.

Integrations

Auto-connect to running services via .nemesis8.toml:

[integrations]
hyperia = true                    # auto-connect if running on port 9800
ferricula = "http://nemesis:8764" # connect to ferricula instance

nemesis8 probes on startup and sets HYPERIA_URL / FERRICULA_URL env vars, forwarded into containers.

Pokeball System

Capture a project, seal it into a hardened Docker image, and run AI agents against it in complete isolation.

nemesis8 pokeball capture https://github.com/user/repo
nemesis8 pokeball seal repo-name
nemesis8 pokeball run repo-name --prompt "analyze the architecture"

Workers are locked down: network=none, read-only rootfs, all capabilities dropped, 4GB RAM, 256 PIDs. AI model access goes through a broker — the container never touches the network.

Capture + Seal

Capture scans a project directory or git URL and generates a pokeball.yaml spec:

nemesis8 pokeball capture ./my-project
nemesis8 pokeball capture https://github.com/org/repo

Auto-detects: language (Python, Rust, Node, Go), package manager, build commands, existing Dockerfiles, monorepo structure, .env files.

Seal combines capture + build in one step:

nemesis8 pokeball seal https://github.com/org/repo

Running a Pokeball

nemesis8 pokeball run repo-name --prompt "fix the failing tests"

This starts a sealed worker container and a broker that talks to the Claude API. The broker sends tool calls (bash, file_read, file_write, grep, glob) to the worker via filesystem messaging. Multi-turn conversation loop until done or timeout.

nemesis8 pokeball list              # list stored pokeballs
nemesis8 pokeball inspect repo-name # show spec details
nemesis8 pokeball remove repo-name  # delete pokeball + image

Pokeball Registry

Share pokeball specs via the registry:

nemesis8 pokeball publish repo-name --description "what it does"
nemesis8 pokeball pull repo-name

Publish creates a PR on the registry repo. Pull downloads the spec to your local store.

SQL Queries

When using ferricula for memory, you can query the docs table directly via ferricula_recall:

# All memories
SELECT id, text, fidelity, recalls FROM docs

# Keystones only
SELECT id, text FROM docs WHERE keystone = 1

# Search text
SELECT id, text FROM docs WHERE text LIKE '%Docker%'

# Code index (from ferricula-code)
SELECT id, text FROM docs WHERE text LIKE '[project:nemesis8]%'

# Most recalled
SELECT id, text, recalls FROM docs ORDER BY recalls DESC LIMIT 10

# Emotion distribution
SELECT emotion_primary, COUNT(*) FROM docs GROUP BY emotion_primary

See the full SQL reference for more examples.
